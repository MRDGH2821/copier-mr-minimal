## 2026-03-08T20:54:31+11:00

### Prompt

> set it up without flake utils

### What Was Done

Rewrote `flake.nix` to remove the `flake-utils` dependency entirely.

#### Changes Made

- Removed `flake-utils` from `inputs`
- Replaced `flake-utils.lib.eachDefaultSystem` with `builtins.currentSystem` to resolve the current system at evaluation time
- Keyed `devShells` directly as `devShells.${system}.default` instead of relying on `flake-utils` to expand per-system outputs
- Restored comments and `shellHook` that were stripped in the previous version

#### Trade-off

Using `builtins.currentSystem` means the flake only exposes the shell for the system it is evaluated on (which is the normal case for a dev shell). If cross-system output enumeration is ever needed, a simple `lib.genAttrs` over an explicit list of systems would be the next step — no external dependency required.
